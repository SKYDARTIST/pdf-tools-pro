The $14B PDF software market is dominated by subscription-heavy, cloud-dependent tools (Adobe, Foxit, SmallPDF) that:
- Force users into $12-30/month perpetual subscriptions
- Upload sensitive documents to cloud servers for AI processing
- Lack transparency about data usage and AI training practices

## 7. Technology Stack Rationale

### Core Technologies

#### Frontend: React 19 + Vite 7
**Why React 19?**
- Industry-standard library with massive developer ecosystem
- New concurrent rendering improves performance for complex PDF UI (500+ page documents)
- Broad hiring pool vs. niche frameworks

**Why Vite over Create React App?**
- 10-20x faster hot module replacement (HMR) speeds up development
- Optimized build output (smaller bundle sizes â†’ faster mobile load times)
- Native ESM support future-proofs the codebase

#### Mobile: Capacitor 8
**Why Capacitor over React Native or Flutter?**
- **Same codebase for Web + Mobile**: ~40% reduction in development time
- **Native WebView rendering**: Leverages browser optimizations for PDF.js
- **Web-first mindset**: Easier to debug (Chrome DevTools work natively)
- **Plugin ecosystem**: Direct access to native APIs (Camera, Filesystem, Share)

**Trade-off Accepted:** Slightly larger app size vs. native apps (~15MB vs. 8MB for pure native)

#### PDF Processing: pdf-lib + pdfjs-dist
**Why pdf-lib?**
- Pure JavaScript implementation (works in browser + mobile WebView)
- Enables offline manipulation (merge, split, rotate) without server
- Active maintenance + MIT license

**Why pdfjs-dist (Mozilla)?**
- Industry-standard PDF renderer (powers Firefox's PDF viewer)
- Handles complex PDF features (forms, annotations, encryption)
- Web Worker support prevents UI blocking

**Trade-off Accepted:** Learning curve for advanced PDF features, but avoids vendor lock-in

#### AI: Google Gemini (2.0 Flash Lite)
**Why Gemini over OpenAI GPT or Claude?**
- **2M token context window**: Allows processing 500+ page documents in single request
- **Cost efficiency**: Flash Lite is 60% cheaper than GPT-4o for similar tasks
- **Vision capabilities**: Enables Smart Redact (detect PII in scanned documents)
- **Enterprise privacy tier**: Data not used for training (critical for privacy promise)

**Trade-off Accepted:** Less "creative" than GPT-4 for open-ended tasks, but superior for structured extraction

#### Backend: Node.js + Express
**Why Node.js?**
- Lightweight proxy layer (only handles signature validation + forwarding)
- JavaScript consistency across frontend/backend (easier team ramp-up)
- Serverless-ready (easy to migrate to Vercel/Cloudflare Workers)

#### Auth & User State: Supabase
**Why Supabase over Firebase or custom auth?**
- **Open-source**: Avoids vendor lock-in (can self-host if needed)
- **PostgreSQL-backed**: Relational data model better for subscription tiers vs. NoSQL
- **Built-in Google OAuth**: Simplifies mobile authentication
- **Row-level security**: Database-level access control (defense in depth)

**Trade-off Accepted:** Smaller ecosystem vs. Firebase, but aligns with privacy-first ethos

## 8. Security & Privacy Architecture

### Privacy-First Design Principles

#### 1. Local Storage Only
- **Documents**: Stored in Capacitor Filesystem (sandboxed app storage)
- **User Data**: Encrypted at rest by OS (iOS Keychain, Android Keystore)
- **Deleted Data**: Overwritten immediately (no "trash" folder)

#### 2. Stateless AI Requests
```javascript
// Example: How AI requests are anonymized
async function sendAIRequest(documentText: string) {
  const payload = {
    text: documentText,  // NO filename, NO user ID
    operation: 'summarize',
    timestamp: Date.now(),
  };

  // HMAC signature prevents unauthorized access
  const signature = await generateHMAC(payload, deviceKey);

  // Send to proxy (no cookies, no tracking headers)
  const response = await fetch('https://proxy.antigravity.app/ai', {
    method: 'POST',
    headers: {
      'X-Signature': signature,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify(payload),
  });

  return response.json();
}
```

```typescript
// Example: How we merge PDFs (simplified)
import { PDFDocument } from 'pdf-lib';

async function mergePDFs(files: File[]): Promise<Uint8Array> {
  const mergedPdf = await PDFDocument.create();

  for (const file of files) {
    const arrayBuffer = await file.arrayBuffer();
    const pdf = await PDFDocument.load(arrayBuffer);
    const pages = await mergedPdf.copyPages(pdf, pdf.getPageIndices());
    pages.forEach(page => mergedPdf.addPage(page));
  }

  // Rebuild XRef table for stability
  return await mergedPdf.save();
}
```

```typescript
// Server-side: Proxy validation
import { createHmac } from 'crypto';

app.post('/ai', async (req, res) => {
  const { prompt, operation, timestamp, signature } = req.body;

  // Validate HMAC signature
  const expectedSig = createHmac('sha256', SERVER_SECRET)
    .update(`${prompt}${operation}${timestamp}`)
    .digest('hex');

  if (signature !== expectedSig) {
    return res.status(401).json({ error: 'Invalid signature' });
  }

  // Return result (no logging)
```

### C. Play Integrity Flow

```typescript
// Client-side: Request integrity token
import { PlayIntegrity } from '@capacitor-community/play-integrity';

async function validatePurchase(purchaseToken: string) {
  // Get device attestation
  const { token } = await PlayIntegrity.requestIntegrityToken({
    nonce: purchaseToken,
  });

  // Send to server for verification
  const response = await fetch('/api/verify-purchase', {
    method: 'POST',
    body: JSON.stringify({ integrityToken: token }),
  });

  return response.ok;
}

// Server-side: Verify integrity token
import { google } from 'googleapis';

async function verifyIntegrityToken(token: string) {
  const playIntegrity = google.playintegrity('v1');

  const result = await playIntegrity.v1.decodeIntegrityToken({
    packageName: 'com.antigravity.pdf',
    body: { integrityToken: token },
  });

  // Check verdict
  const { appIntegrity, deviceIntegrity, accountDetails } = result.data.tokenPayloadExternal;

  if (appIntegrity?.appRecognitionVerdict !== 'PLAY_RECOGNIZED') {
    throw new Error('App not recognized by Play Store');
  }

  if (deviceIntegrity?.deviceRecognitionVerdict?.includes('MEETS_DEVICE_INTEGRITY') === false) {
    throw new Error('Device is rooted or emulated');
  }

  return true;
}
```

- Prevents modified APKs from accessing AI features
- Links purchase to legitimate device (can't share account)
